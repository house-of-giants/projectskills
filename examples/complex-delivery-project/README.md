# Complex Delivery Project Example

A public-safe fake example showing the full Project Skills chain on a messier delivery project.

This example includes:

- a local project context generated from snapshots rather than live systems
- accepted decisions with canonical IDs
- candidate decisions and open questions
- stale specification language superseded by later decisions
- requirements with `Requirement ready` statuses
- proposed requirements from candidates using `PREQ-*`
- execution trace rows that distinguish implementation evidence from client acceptance
- non-goals and constraints tracked as scope guards

Files:

- `project-context.md`
- `decisions.md`
- `requirements.md`
- `execution-trace.md`
